### Conventions

- **Naming**: `camelCase` for functions and variables; `PascalCase` for types and exported plugin constants; `kebab-case` for file names and the plugin directory name.
- **Module organization**: Thin entry point (`index.ts`) re-exports the plugin function. Business logic split into focused single-responsibility files. No barrel re-export gymnastics.
- **Test patterns**: Bun built-in test runner (`bun test`). Test files named `*.test.ts`, co-located in `src/` or in a top-level `test/` dir. No mocking framework needed — pure unit tests on stateless functions with injected timestamps.
- **Config patterns**: All thresholds are compile-time constants in `types.ts`. No `.env`, no runtime config files — constants are explicit and centralized.

### Reusable Abstractions

- `@opencode-ai/plugin` — `Plugin`, `PluginInput`, `Hooks` types. Import directly; do not re-declare.
- `@opencode-ai/sdk` — `Event`, `EventSessionIdle` types (transitive via plugin SDK).
- Bun `console.error` / `console.log` — standard logging; no logger library.
- Bun `Date.now()` — wall-clock timestamps throughout; no external time library.

### Tooling Constraints

- No global `tsc` — TypeScript type-checking must run via `bunx tsc --noEmit` using the locally installed typescript from `node_modules`.
- ESM only — no `require()`, no CJS exports, no `.cjs` extensions.
- No external dependencies beyond `@opencode-ai/plugin` — the PRD explicitly mandates this.
- Plugin registered in `~/.config/opencode/opencode.jsonc` by adding the absolute path to the `"plugin"` array.

### Patterns to Avoid

- Do **not** use `export default` for the plugin — OpenCode loads named exports (`export const PluginName: Plugin = ...`).
- Do **not** access `event.idleTime` or `event.sessionId` — the real SDK uses `event.properties.sessionID` with no `idleTime` field. The PRD is wrong about these.
- Do **not** return `config: () => ({ tools: [], agents: [] })` from the `Hooks` object — the `config` hook has a different signature and this shape is not valid.
- Do **not** use CommonJS `module.exports`.
- Do **not** introduce class hierarchies or dependency injection containers.

### Research Gaps

1. **`session.idle` firing frequency**: Not documented — unclear whether OpenCode fires `session.idle` once when idle begins, or repeatedly on an interval. The implementation must be defensive: track the time of first idle detection and use wall-clock delta, not a counter of event firings.

2. **`session.promptAsync` vs `session.prompt`**: The async variant (HTTP POST `/session/:id/prompt_async`, 204 no-content) is the correct choice for fire-and-forget injection. The SDK method name `client.session.promptAsync()` matches the OpenAPI spec naming convention and is confirmed by the `SessionPromptAsyncData` type. However, the actual runtime behavior (whether it triggers a full AI response cycle) was not observable in static analysis. **Implementers should test this manually** in Step 3 validation.

3. **`session.status` event**: The SDK has `EventSessionStatus` with a `status` field (`{ type: "idle" } | { type: "busy" } | ...`). It's unclear whether `session.idle` is a distinct event or an alias for `session.status` with `status.type === "idle"`. The `event.type === "session.idle"` check from the PRD is confirmed as a real event type in the SDK types, so treat it as distinct.

4. **Cleanup on session end**: No `session.end` or `session.close` event was found in the type definitions. Session state stored in a `Map` will grow unbounded in theory. Given the in-process nature of plugins and typical session counts, this is acceptable for v1. Flag for future cleanup hook if OpenCode adds one.

**Code changes to apply**:

```typescript
// src/throttle.ts

import type { SessionState } from "./types.js"
import {
  COOLDOWN_PERIOD,
  MAX_HOURLY_CONTINUES,
  ONE_HOUR,
  sessionStates,
} from "./types.js"

export function getOrCreateState(sessionID: string): SessionState {
  if (!sessionStates.has(sessionID)) {
    sessionStates.set(sessionID, {
      lastContinuation: 0,
      hourlyCount: 0,
      hourStart: 0,
      lastIdleSeen: 0,
      lastUserMessage: 0,
    })
  }
  return sessionStates.get(sessionID)!
}

export function canContinue(state: SessionState, now: number): boolean {
  // Cooldown: must be at least COOLDOWN_PERIOD since last continuation
  if (state.lastContinuation > 0 && now - state.lastContinuation < COOLDOWN_PERIOD) {
    return false
  }
  // Hourly cap: reset window if more than an hour has passed
  const inCurrentWindow = state.hourStart > 0 && now - state.hourStart < ONE_HOUR
  if (inCurrentWindow && state.hourlyCount >= MAX_HOURLY_CONTINUES) {
    return false
  }
  return true
}

export function recordContinuation(state: SessionState, now: number): void {
  state.lastContinuation = now
  // Reset hourly window if expired
  if (state.hourStart === 0 || now - state.hourStart >= ONE_HOUR) {
    state.hourStart = now
    state.hourlyCount = 0
  }
  state.hourlyCount++
}
```

// src/throttle.test.ts

import { describe, it, expect, beforeEach } from "bun:test"
import { canContinue, recordContinuation, getOrCreateState } from "./throttle.js"
import { sessionStates, COOLDOWN_PERIOD, MAX_HOURLY_CONTINUES, ONE_HOUR } from "./types.js"
import type { SessionState } from "./types.js"

describe("canContinue", () => {
  let state: SessionState

  beforeEach(() => {
    sessionStates.clear()
    state = getOrCreateState("test-session")
  })

  it("allows first continuation (no history)", () => {
    expect(canContinue(state, Date.now())).toBe(true)
  })

  it("blocks within cooldown window", () => {
    const now = 1000000
    recordContinuation(state, now)
    expect(canContinue(state, now + COOLDOWN_PERIOD - 1)).toBe(false)
  })

  it("allows after cooldown expires", () => {
    const now = 1000000
    recordContinuation(state, now)
    expect(canContinue(state, now + COOLDOWN_PERIOD)).toBe(true)
  })

  it("blocks when hourly cap reached within window", () => {
    const now = 1000000
    for (let i = 0; i < MAX_HOURLY_CONTINUES; i++) {
      recordContinuation(state, now + i * COOLDOWN_PERIOD)
    }
    expect(canContinue(state, now + MAX_HOURLY_CONTINUES * COOLDOWN_PERIOD)).toBe(false)

    const now = 1000000
    for (let i = 0; i < MAX_HOURLY_CONTINUES; i++) {
      recordContinuation(state, now + i * COOLDOWN_PERIOD)

    // One hour + cooldown later
    expect(canContinue(state, now + ONE_HOUR + COOLDOWN_PERIOD)).toBe(true)

**Code changes to apply**:

```typescript
// src/idle-handler.ts

import type { Event } from "@opencode-ai/sdk"
import type { PluginInput } from "@opencode-ai/plugin"
import {
  IDLE_THRESHOLD,
  CONTINUE_PROMPT,
} from "./types.js"
import { getOrCreateState, canContinue, recordContinuation } from "./throttle.js"

export async function handleIdleEvent(
  { event }: { event: Event },
  client: PluginInput["client"]
): Promise<void> {
  if (event.type !== "session.idle") return

  const sessionID = event.properties.sessionID
  const state = getOrCreateState(sessionID)
  const now = Date.now()

  // Record when we first saw this idle state
  if (state.lastIdleSeen === 0) {
    state.lastIdleSeen = now
    return  // Not idle long enough yet — wait for next event
  }

  // Check if idle threshold has been reached
  if (now - state.lastIdleSeen < IDLE_THRESHOLD) return

  // Check throttle
  if (!canContinue(state, now)) return

  try {
    await client.session.promptAsync({
      path: { id: sessionID },
      body: {
        parts: [{ type: "text", text: CONTINUE_PROMPT }],
      },
    })
    recordContinuation(state, now)
    state.lastIdleSeen = 0  // Reset: next idle event starts a fresh window
  } catch (err) {
    console.error("[auto-continue] Failed to inject continuation prompt:", err)
    // Do not rethrow — a failed prompt must not crash the plugin
  }
}

export function handleUserMessage(input: { sessionID: string }): void {
  const state = getOrCreateState(input.sessionID)
  state.lastIdleSeen = 0
  state.lastUserMessage = Date.now()
}
```

```typescript
// src/idle-handler.test.ts

import { describe, it, expect, beforeEach, mock } from "bun:test"
import { handleIdleEvent, handleUserMessage } from "./idle-handler.js"
import { sessionStates, IDLE_THRESHOLD, COOLDOWN_PERIOD } from "./types.js"
import { getOrCreateState } from "./throttle.js"

// Minimal mock client — only the method we use
function makeClient(promptFn = mock(() => Promise.resolve())) {
  return {
    session: {
      promptAsync: promptFn,
    },
  } as any
}

const idleEvent = (sessionID: string) => ({
  event: { type: "session.idle" as const, properties: { sessionID } },
})

describe("handleIdleEvent", () => {
  beforeEach(() => sessionStates.clear())

  it("ignores non-idle events", async () => {
    const client = makeClient()
    await handleIdleEvent(
      { event: { type: "session.status", properties: { sessionID: "s1", status: { type: "idle" } } } as any },
      client as any
    )
    expect(client.session.promptAsync).not.toHaveBeenCalled()
  })

  it("records lastIdleSeen on first idle event and does not prompt yet", async () => {
    const client = makeClient()
    await handleIdleEvent(idleEvent("s1"), client as any)
    expect(client.session.promptAsync).not.toHaveBeenCalled()
    const state = getOrCreateState("s1")
    expect(state.lastIdleSeen).toBeGreaterThan(0)
  })

  it("does not prompt if idle threshold not yet reached", async () => {
    const client = makeClient()
    // First event: records lastIdleSeen
    await handleIdleEvent(idleEvent("s2"), client as any)
    const state = getOrCreateState("s2")
    // Manually backdate by less than threshold
    state.lastIdleSeen = Date.now() - (IDLE_THRESHOLD - 1000)
    await handleIdleEvent(idleEvent("s2"), client as any)
    expect(client.session.promptAsync).not.toHaveBeenCalled()
  })

  it("prompts when idle threshold is exceeded", async () => {
    const promptFn = mock(() => Promise.resolve())
    const client = makeClient(promptFn)
    await handleIdleEvent(idleEvent("s3"), client as any)
    const state = getOrCreateState("s3")
    // Backdate past threshold
    state.lastIdleSeen = Date.now() - IDLE_THRESHOLD - 1000
    await handleIdleEvent(idleEvent("s3"), client as any)
    expect(promptFn).toHaveBeenCalledTimes(1)

# opencode-auto-continue

An OpenCode plugin that nudges the AI to continue work after 5 minutes of session idle time.

## Installation

1. Clone this repo.
2. Run `bun install` inside `auto-continue-plugin/`.
3. Register the plugin in `~/.config/opencode/opencode.jsonc`:

```json
{
  "plugin": ["/absolute/path/to/auto-continue-plugin/src/index.ts"]
}
```

## How it works

- Listens for `session.idle` events from OpenCode.
- Tracks how long the session has been idle using wall-clock time.
- After **5 minutes** of continuous idle, injects a single continuation prompt.
- The AI receives: *"Please assess if there's any additional work needed and continue if appropriate."*
- If the AI decides there is nothing left to do, it says so and stops.
- Any new user message resets the idle timer.

## Rate limits

| Limit | Value |
|---|---|
| Idle threshold before first prompt | 5 minutes |
| Minimum gap between prompts | 10 minutes |
| Maximum prompts per hour | 3 |

## Caveats

- State is in-memory. Restarting OpenCode resets all counters.
- No configuration UI — edit `src/types.ts` constants to change thresholds.
